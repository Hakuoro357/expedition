CHANGES NOT MADE (and reasons):
- The codex review suggested adding an `off` method to SdkService;
  instead we used the singleton-context pattern which avoids the
  off-needed problem. GP SDK does not document a clean off-token,
  so a single global listener is a stronger guarantee than per-scene
  on/off.
- Tests for the actual SDK Promise rejection path are not added at
  the gate level — try/catch + async/await pattern in
  GamePushSdkService is well-known-safe and would require mocking
  GP SDK in unit tests, which we don't have infra for. The
  socialsContext tests cover the riskier listener-attribution path.
