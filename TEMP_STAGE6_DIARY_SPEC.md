# Solitaire: Expedition — Stage 6 Spec

## Тема

`DiaryScene -> narrative progression`

## Статус

`Временный рабочий файл`

- Это узкий spec только для шага перестройки дневника.
- После внедрения его можно удалить.

## Цель этапа

- Перестать держать `DiaryScene` только как экран коллекции.
- Начать показывать story progression через открытые записи.
- Сохранить коллекцию артефактов и прогресс по главам.

## Что входит

- helper для narrative entry display
- минимальная перестройка `DiaryScene`

## Что не входит

- полноценный scrollable codex
- сложная фильтрация по авторам
- отдельный экран записи
- редизайн всей меты

## Целевой результат

`DiaryScene` должен показывать:
- latest unlocked entry
- счётчик открытых записей
- краткий список последних открытых записей
- коллекцию артефактов
- прогресс по главам

## Ограничения

- используем текущий размер сцены и существующий UI shell
- без новой save migration
- unlocked entries вычисляем через `completedNodes -> node.entryId`

## Критерии готовности

- игрок видит, что дневник рассказывает историю
- история читается по progression, а не только через docs
- artifacts остаются на экране
- tests/build остаются зелёными
