# Archive Redesign

## Статус

`Approved`

## Задача

Переделать текущий экран `Дневник` в новый экран `Архив`, который не дублирует главный маршрутный экран и работает только как архив уже открытого контента.

## Роль экрана

`Архив` нужен для:

- перечитывания открытых записей;
- просмотра найденных артефактов;
- повторного открытия больших карточек записи и артефакта.

`Архив` не нужен для:

- первого reveal записи;
- первого reveal артефакта;
- навигации по маршруту.

## Общая структура

Экран состоит из:

1. верхней шапки `Архив`;
2. вкладок:
   - `Записи`
   - `Артефакты`
3. основной области контента;
4. постоянной нижней плашки в том же формате, что и на главном экране.

Жёсткая рамка / бордюр вокруг экрана не используются. Контентная область живёт как единое поле без обводки, а нижняя плашка отделяется только сменой тона и тонкой верхней линией.

Фон `Архива` остаётся нейтральным и чистым, без полосок маршрута и без графического шума.

## Нижняя плашка

Плашка идентична главному экрану по формату:

- только иконки;
- те же три действия:
  - `На главную`
  - `Маршрут дня`
  - `Настройки`
- у первой кнопки отдельная иконка `дом`, а не иконка `Архива`;
- активного состояния у кнопки возврата нет, потому что она ведёт на другой экран.
- нижняя плашка визуально стоит отдельно от контента, а не внутри рамки экрана.

Отдельной кнопки возврата в шапке нет. Возврат к основному маршруту делается через иконку `дом` в нижней плашке.

## Вкладка `Записи`

### Общий вид

Показывает вертикальный список открытых записей.

Каждая запись в списке содержит:

- портрет автора;
- название этапа маршрута;
- имя автора;
- короткий excerpt;
- визуальное состояние `открыто`.

### Поведение

- по нажатию на запись открывается крупная модалка полного текста;
- модалка использует тот же narrative-текст, что и reward detail;
- запись открывается без перехода на другой экран.

## Вкладка `Артефакты`

### Общий вид

Показывает сетку только открытых артефактов.

Закрытые артефакты:

- не показываются;
- не считаются;
- не блюрятся;
- не маркируются замком.

### Поведение

- по нажатию на открытый артефакт открывается большая карточка артефакта;
- карточка остаётся модальной поверх `Архива`.

## Модалки

### Модалка записи

Содержит:

- название этапа маршрута;
- портрет автора;
- имя автора;
- полный текст записи;
- кнопку закрытия.

### Карточка артефакта

Сохраняет текущую модель:

- название артефакта;
- большая картинка;
- короткое описание;
- кнопка закрытия.

## Что уходит из текущего экрана

Из старого `Дневника` убираются:

- chapter progress как главный контент;
- старый overview-блок latest/recent как основной сценарий;
- кнопка `На карту` как отдельный старый CTA.

## Приоритеты UX

1. Архив должен ощущаться как спокойный просмотрщик, а не как второй meta-hub.
2. Вкладки должны быть простыми и очевидными.
3. Записи должны читаться легче, чем на старом экране.
4. Артефакты должны сохранить коллекционное ощущение.

## Scope

Внутри задачи:

- переименование экрана в `Архив`;
- новая структура с двумя вкладками;
- перенос нижней плашки на новый формат;
- список записей с портретами;
- сохранение артефактной карточки;
- модалка полного текста записи.

Вне scope:

- финальные портретные ассеты персонажей;
- дополнительные фильтры и сортировки;
- поиск по архиву.

## Detail-страницы вместо модалок

В `Архиве` модалки больше не используются.

Новая логика:

- клик по записи ведёт на общую detail-страницу типа `Запись`;
- клик по артефакту ведёт на общую detail-страницу типа `Артефакт`;
- если у сущности есть связанная пара `Запись / Артефакт`, переключение происходит внутри одной и той же detail-страницы.

## Единый layout

`Архив`, `Маршрут`, `Настройки`, `Запись` и `Артефакт` должны использовать один и тот же layout-принцип:

- без жёсткого бордюра;
- с одинаковой нижней плашкой;
- с одинаковой логикой safe-area;
- с общей типографической системой;
- с максимальным использованием вертикали под основной контент.

## Detail-кнопка

На detail-страницах `Запись` и `Артефакт` верхняя сервисная кнопка:

- называется `Назад`;
- ведёт в `Архив`;
- не ведёт на главный маршрут.
