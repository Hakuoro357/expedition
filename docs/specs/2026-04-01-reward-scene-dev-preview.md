# RewardScene Dev Preview

## Статус

`Approved`

## Задача

Добавить безопасный dev-only способ открыть `RewardScene` напрямую по URL, не проходя партию вручную.

## Цель

- быстро проверять layout и reveal-flow;
- открывать `RewardScene` с нужной точкой и режимом;
- не добавлять временные debug-кнопки в обычный UI;
- не менять продуктовый сценарий для игрока.

## Формат вызова

Прямой вход должен работать по URL-параметрам:

`?preview=reward&dealId=c1n3&mode=adventure`

Допустимые параметры:

- `preview=reward`
- `dealId=<id точки>`
- `mode=adventure | daily | quick`

## Правила

- hook работает только в dev-режиме;
- в production-сборке прямой вход недоступен;
- обычный запуск игры без параметров не меняется;
- если `dealId` отсутствует или некорректен, используется безопасный fallback:
  - `mode=quick`, если нельзя корректно восстановить точку;
- решение должно быть локальным и легко удаляемым.

## Точка входа

Предпочтительно обрабатывать preview как можно раньше:

- либо в `BootScene`,
- либо до старта основной сцены, если это можно сделать чище.

## Ожидаемый результат

Разработчик может открыть локально:

`http://127.0.0.1:4175/?preview=reward&dealId=c1n3&mode=adventure`

и сразу попасть на `RewardScene`.

## Dev Preview Hub

Дополнительно нужен dev-only хаб готовых reward-preview сценариев.

Формат вызова:

`?preview=reward-list`

Требования:

- хаб доступен только в dev-режиме;
- он показывает короткий список готовых ссылок на reward preview;
- каждая ссылка должна открывать отдельный сценарий в новой вкладке или по прямому URL;
- хаб не должен менять save, cloud save или обычный пользовательский flow.

Минимальный набор сценариев:

- `c1n3` — запись + карта
- `c2n6` — запись + артефакт
- `c3n1` — запись + карта

Допустимо добавить ещё 2-3 полезных preset-сценария, если они реально помогают QA.
