## Принципы

- Код не затрагивается.
- Сначала готовятся ассеты, критичные для релиза.
- По возможности избегаем текста внутри растровой графики.
- Все ассеты должны поддерживать атмосферу `спокойной пропавшей экспедиции`.
- Приоритет у читаемости, а не у декоративности.

## 1. Promo-ассеты

### 1.1 Иконка игры

- Приоритет: `P0`
- Назначение: каталог и рекламные материалы Яндекс Игр
- Формат: `PNG`
- Размер: `512x512`
- Текст: не использовать
- Сюжетная идея: карты + навигационный диск + намёк на экспедицию

### 1.2 Maskable icon

- Приоритет: `P0`
- Назначение: маскируемая иконка
- Формат: `PNG`
- Размер: `512x512`
- Текст: не использовать
- Особенность: главный объект в безопасной центральной зоне

### 1.3 Cover

- Приоритет: `P0`
- Назначение: основная обложка игры
- Формат: `PNG`
- Размер: `800x470`
- Текст: лучше без текста, либо с легко заменяемым слоем
- Сюжетная идея: пасьянс + карта + экспедиционный мотив

### 1.4 Hero image

- Приоритет: `P1`
- Назначение: спецподборки и промо
- Формат: `PNG` или `JPG`
- Размер: `1560x520`
- Текст: минимальный или отдельным слоем

### 1.5 Скриншоты для карточки игры

- Приоритет: `P0`
- Назначение: страница игры в каталоге
- Формат: `PNG` или `JPG`
- Типы: `mobile`, `desktop`
- Количество: минимум `4-6`
- Сцены:
- карта экспедиции;
- основной стол;
- экран дневника;
- экран награды;
- момент победы;
- коллекция находок.

## 8. Технические требования к именованию

- все имена файлов на латинице;
- без пробелов;
- без кириллицы;
- единый стиль: `snake_case`;
- версии отмечать суффиксами: `_v01`, `_v02`;
- для языковых вариантов использовать суффиксы вида `_en`, `_tr`, `_de`.
