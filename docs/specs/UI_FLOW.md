# Solitaire: Expedition — UI Flow

## Цель

Зафиксировать минимальный набор экранов `v1.0`, переходы между ними и порядок показа в первой сессии.

## Главный принцип

Игрок должен доходить до первой партии максимально быстро.

Целевой путь:

`Запуск -> Карта экспедиции -> Играть -> Партия -> Экран результата точки -> Карта`

## Принятое решение по рендеру текста

Для `v1.0` принят гибридный подход:

- текстовые UI-блоки и narrative-блоки можно рендерить через `DOM`, если от этого критически выигрывает читаемость;
- игровую графику, маршрут, карты, расклад, hit areas, анимации и прочую интерактивную часть держать в `Phaser`.

Причина:

- игра будет мультиязычной;
- кроме `ru` и `en` ожидаются и более сложные локали, включая `zh`;
- browser DOM рендерит длинные текстовые блоки и сложные шрифтовые наборы заметно чище и стабильнее, чем canvas-текст `Phaser`.
- для стабильного layout такой текстовый слой должен быть привязан к реальному `canvas rect`, а не жить как независимый потоковый блок.

Практическое правило:

- `DOM` — для заголовков экранов, narrative-текста, subtitle, chapter headers, reward text blocks, diary text blocks, settings text blocks;
- `Phaser` — для расклада, route graphics, node circles, progress bars, background art, FX и интерактивной геометрии.

На текущем этапе разрешён смешанный экран:

- верхний текстовый блок через `DOM`;
- всё остальное на том же экране через `Phaser`.

Техническое правило:

- если нужен чёткий крупный текст, используем внешний overlay поверх контейнера приложения;
- overlay масштабируем от логического размера игры и синхронизируем с `canvas.getBoundingClientRect()`;
- `Phaser DOMElement` не использовать как основной layout-механизм для больших текстовых блоков на `FIT`-скейле.

## Список экранов `v1.0`

### 1. Загрузка

Назначение:

- инициализация игры;
- инициализация SDK;
- загрузка сохранения;
- переход в нужную точку входа.

Переходы:

- в `Карту экспедиции`, если есть прогресс;
- в `Первый вход`, если игрок новый.

### 2. Первый вход

Назначение:

- быстро показать тему игры;
- не перегрузить пользователя обучением.

Содержимое:

- логотип;
- короткий подзаголовок;
- кнопка `Начать`.

Переходы:

- в `Карту экспедиции`.

### 3. Карта экспедиции

Назначение:

- главный хаб игры;
- выбор доступной точки маршрута;
- вход в ежедневный режим;
- вход в дневник.

Содержимое:

- текущая глава;
- цепочка точек маршрута;
- текущий прогресс;
- кнопка `Играть`;
- кнопка `Daily`;
- кнопка `Дневник`;
- мини-блок валюты;
- кнопка `Настройки`.

Рендер:

- title, subtitle, название экспедиции и chapter header можно рендерить через `DOM`;
- маршрут, точки, прогресс-бар и кнопочные hit areas остаются в `Phaser`.

Переходы:

- в `Экран партии`;
- в `Daily Route`;
- в `Дневник`;
- в `Настройки`.

### 4. Экран партии

Назначение:

- основная игровая сессия.

Содержимое:

- игровое поле `Klondike`;
- верхняя панель;
- кнопки `Undo`, `Hint`, `Menu`;
- индикатор текущего уровня / daily;
- при необходимости мягкая подсветка обучения.

Переходы:

- в `Победу`;
- в `Поражение / Рестарт`;
- в `Пауза`.

### 5. Пауза

Назначение:

- выйти из партии без потери контекста;
- дать базовые настройки.

Содержимое:

- `Продолжить`;
- `Рестарт`;
- `На карту`;
- звук / музыка.

Переходы:

- обратно в `Экран партии`;
- в `Карту экспедиции`.

### 6. Экран результата точки

Назначение:

- показать итог победы;
- сразу раскрыть новые награды точки;
- дать optional-доступ к подробностям без перехода в дневник;
- предложить рекламный бонус как вторичный сервисный элемент.

Содержимое:

- заголовок победы;
- монеты;
- прогресс главы;
- блок `Вы нашли`;
- карточки новых наград текущей точки;
- кнопка `Продолжить`;
- кнопка `Смотреть рекламу`, если она доступна.

Reveal-логика:

- `Вы нашли` показывает только новые награды этой точки;
- основные игроковые типы наград:
  - запись дневника
  - артефакт
  - фрагмент карты
- по клику на карточку открывается detail-модалка;
- открывать detail-модалки не обязательно.

Рендер:

- заголовок, summary, блок `Вы нашли` и detail-модалки предпочтительно рендерить через `DOM`;
- фон, панель, hit areas, графику мини-карты и кнопочные зоны оставлять в `Phaser`.

Переходы:

- обратно в `Карту экспедиции`.

### 7. Поражение / Рестарт

Назначение:

- мягко вернуть игрока в цикл без раздражения.

Содержимое:

- причина завершения;
- `Попробовать еще раз`;
- `На карту`;
- при необходимости rewarded-опция.

Переходы:

- в `Экран партии`;
- в `Карту экспедиции`.

### 8. Daily Route

Назначение:

- отдельный daily-цикл возврата.

Содержимое:

- текущий daily;
- награда дня;
- статус `пройден / не пройден`;
- кнопка `Играть`.

Переходы:

- в `Экран партии`;
- после завершения обратно в `Daily Route` или `Карту экспедиции`.

### 9. Дневник

Назначение:

- показать прогресс меты.

Содержимое:

- артефакты;
- завершенные главы;
- серия входов;
- косметические разблокировки.

Рендер:

- narrative entries, chapter summaries и коллекционные подписи можно рендерить через `DOM`;
- label кнопки назад можно рендерить через `DOM`, оставляя hit area в `Phaser`;
- фон, декоративные рамки, иконки и интерактивные области оставлять в `Phaser`.

Переходы:

- в `Карту экспедиции`.

### 10. Настройки

Назначение:

- базовый сервисный экран.

Содержимое:

- язык;
- звук;
- музыка;
- политика / служебные ссылки, если нужны.

Рендер:

- текст и простые сервисные блоки можно рендерить через `DOM`, если это улучшает читаемость и упрощает мультиязычную вёрстку.

Переходы:

- в `Карту экспедиции`;
- из `Паузы` обратно в `Экран партии`.

## Первый пользовательский сценарий

### Первая сессия

1. Игрок открывает игру.
2. Видит короткий экран первого входа.
3. Попадает на первую точку карты.
4. Нажимает `Играть`.
5. Проходит первую партию с мягкими подсказками.
6. Видит экран результата точки с блоком `Вы нашли`.
7. При желании открывает подробности награды.
8. Видит, что маршрут продвинулся.
9. Получает мотивацию начать следующую партию.

## Основные переходы

### Основной цикл

`Загрузка -> Карта -> Партия -> Экран результата точки -> Карта`

### Цикл проигрыша

`Партия -> Поражение -> Рестарт -> Партия`

### Ежедневный цикл

`Карта -> Daily -> Партия -> Экран результата точки -> Карта`

### Мета-цикл

`Карта -> Дневник -> Карта`

## Что нельзя допустить

- длинного стартового интро;
- лишних модалок до первой партии;
- рекламы до первого завершенного расклада;
- ситуации, где игрок не понимает, куда нажать дальше;
- тяжелого меню с десятком второстепенных кнопок.

## Приоритет экранов

### Обязательные для `v1.0`

- загрузка;
- карта;
- партия;
- победа;
- поражение / рестарт;
- экран результата точки;
- daily;
- дневник;
- настройки.

### Можно упростить

- первый вход можно свести к одному экрану;
- паузу можно сделать минимальной;
- detail-модалки наград можно упрощать, но сам блок `Вы нашли` должен оставаться первым местом reveal.
