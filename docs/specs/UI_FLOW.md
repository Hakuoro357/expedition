# Solitaire: Expedition — UI Flow

## Цель

Зафиксировать минимальный набор экранов `v1.0`, переходы между ними и порядок показа в первой сессии.

## Главный принцип

Игрок должен доходить до первой партии максимально быстро.

Целевой путь:

`Запуск -> Карта экспедиции -> Играть -> Партия -> Победа -> Награда -> Карта`

## Список экранов `v1.0`

### 1. Загрузка

Назначение:

- инициализация игры;
- инициализация SDK;
- загрузка сохранения;
- переход в нужную точку входа.

Переходы:

- в `Карту экспедиции`, если есть прогресс;
- в `Первый вход`, если игрок новый.

### 2. Первый вход

Назначение:

- быстро показать тему игры;
- не перегрузить пользователя обучением.

Содержимое:

- логотип;
- короткий подзаголовок;
- кнопка `Начать`.

Переходы:

- в `Карту экспедиции`.

### 3. Карта экспедиции

Назначение:

- главный хаб игры;
- выбор доступной точки маршрута;
- вход в ежедневный режим;
- вход в дневник.

Содержимое:

- текущая глава;
- цепочка точек маршрута;
- текущий прогресс;
- кнопка `Играть`;
- кнопка `Daily`;
- кнопка `Дневник`;
- мини-блок валюты;
- кнопка `Настройки`.

Переходы:

- в `Экран партии`;
- в `Daily Route`;
- в `Дневник`;
- в `Настройки`.

### 4. Экран партии

Назначение:

- основная игровая сессия.

Содержимое:

- игровое поле `Klondike`;
- верхняя панель;
- кнопки `Undo`, `Hint`, `Menu`;
- индикатор текущего уровня / daily;
- при необходимости мягкая подсветка обучения.

Переходы:

- в `Победу`;
- в `Поражение / Рестарт`;
- в `Пауза`.

### 5. Пауза

Назначение:

- выйти из партии без потери контекста;
- дать базовые настройки.

Содержимое:

- `Продолжить`;
- `Рестарт`;
- `На карту`;
- звук / музыка.

Переходы:

- обратно в `Экран партии`;
- в `Карту экспедиции`.

### 6. Победа

Назначение:

- показать результат партии;
- выдать награды;
- предложить рекламный бонус без давления.

Содержимое:

- короткий экран победы;
- базовая награда;
- опция `Смотреть рекламу` для бонуса;
- кнопка `Продолжить`.

Переходы:

- в `Экран награды`;
- затем в `Карту экспедиции`.

### 7. Поражение / Рестарт

Назначение:

- мягко вернуть игрока в цикл без раздражения.

Содержимое:

- причина завершения;
- `Попробовать еще раз`;
- `На карту`;
- при необходимости rewarded-опция.

Переходы:

- в `Экран партии`;
- в `Карту экспедиции`.

### 8. Экран награды

Назначение:

- визуализировать прогресс после победы.

Содержимое:

- монеты;
- фрагмент артефакта;
- открытая точка;
- новый фон / рубашка, если есть крупная награда.

Переходы:

- в `Карту экспедиции`.

### 9. Daily Route

Назначение:

- отдельный daily-цикл возврата.

Содержимое:

- текущий daily;
- награда дня;
- статус `пройден / не пройден`;
- кнопка `Играть`.

Переходы:

- в `Экран партии`;
- после завершения обратно в `Daily Route` или `Карту экспедиции`.

### 10. Дневник

Назначение:

- показать прогресс меты.

Содержимое:

- артефакты;
- завершенные главы;
- серия входов;
- косметические разблокировки.

Переходы:

- в `Карту экспедиции`.

### 11. Настройки

Назначение:

- базовый сервисный экран.

Содержимое:

- язык;
- звук;
- музыка;
- политика / служебные ссылки, если нужны.

Переходы:

- в `Карту экспедиции`;
- из `Паузы` обратно в `Экран партии`.

## Первый пользовательский сценарий

### Первая сессия

1. Игрок открывает игру.
2. Видит короткий экран первого входа.
3. Попадает на первую точку карты.
4. Нажимает `Играть`.
5. Проходит первую партию с мягкими подсказками.
6. Получает первую награду.
7. Видит, что маршрут продвинулся.
8. Получает мотивацию начать следующую партию.

## Основные переходы

### Основной цикл

`Загрузка -> Карта -> Партия -> Победа -> Награда -> Карта`

### Цикл проигрыша

`Партия -> Поражение -> Рестарт -> Партия`

### Ежедневный цикл

`Карта -> Daily -> Партия -> Награда -> Карта`

### Мета-цикл

`Карта -> Дневник -> Карта`

## Что нельзя допустить

- длинного стартового интро;
- лишних модалок до первой партии;
- рекламы до первого завершенного расклада;
- ситуации, где игрок не понимает, куда нажать дальше;
- тяжелого меню с десятком второстепенных кнопок.

## Приоритет экранов

### Обязательные для `v1.0`

- загрузка;
- карта;
- партия;
- победа;
- поражение / рестарт;
- награда;
- daily;
- дневник;
- настройки.

### Можно упростить

- первый вход можно свести к одному экрану;
- паузу можно сделать минимальной;
- экран награды можно встроить в экран победы, если сроки горят.
