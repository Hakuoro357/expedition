# Route Sheet UX Redesign

## Статус

`Approved`

## Задача

Перестроить meta-UX игры:

- убрать фрагменты карты как отдельный тип награды и как центральный meta-экран;
- заменить карту на `листы маршрута экспедиции`;
- сделать точки маршрута главным входом в прохождение, записи и повторный просмотр найденного;
- переименовать `Дневник` в `Архив` и сузить его роль до архива уже открытого контента.

## Основная идея

Главный экран игры больше не показывает абстрактную карту. Вместо неё игрок видит страницу маршрута экспедиции, где точки расположены по траектории пути и читаются как этапы реального движения.

Точка маршрута теперь объединяет в себе:

- этап пасьянса;
- narrative-узел;
- архивную запись;
- возможный артефакт.

## Главный экран: Маршрут

### Общая структура

- это новый основной экран игры;
- маршрут занимает почти всю поверхность экрана;
- бренд `Solitaire: Expedition` с экрана убирается;
- путь читается `снизу вверх`;
- нижняя панель навигации видна всегда.
- жёсткая рамка / листовой бордюр не используются;
- экран собирается как единое поле без обводки;
- нижняя навигация отделяется от поля только сменой тона и тонкой верхней линией.
- фон маршрута остаётся чистым, без тетрадных полос / строк;
- каждый лист маршрута должен поддерживать свой собственный фон или фоновую тему.

### Листы маршрута

Маршрут делится на `4` листа:

- лист 1: `8` точек;
- лист 2: `8` точек;
- лист 3: `7` точек;
- лист 4: `7` точек.

Это UI-разбиение. Narrative-главы при этом остаются текущими `3`, без пересборки story-структуры.

### Фоны листов

- у каждого листа может быть свой визуальный фон;
- фон не должен мешать читаемости точек и маршрута;
- общий принцип: спокойный тематический бэкграунд без графического шума и без полосок;
- смена фона между листами должна помогать ощущению продвижения по маршруту.

### Навигация между листами

- на мобиле: свайп по горизонтали;
- на десктопе: кнопки по левому и правому краю экрана;
- листать можно только среди уже открытых листов;
- новый лист открывается только после полного прохождения предыдущего листа.

## Состояния точек маршрута

### Пройденная точка

- визуально чистая и полностью читаемая;
- интерактивна;
- по нажатию открывает модалку точки.

### Текущая следующая точка

- единственная доступная непройденная точка;
- самая заметная на листе;
- по нажатию запускает партию;
- рядом с ней показывается короткое описание следующей цели.

### Дальнейшие точки

- визуально присутствуют на маршруте;
- не интерактивны;
- приглушены туманом / силуэтом, без полной читаемости.

## Модалка точки

### Общий принцип

Клик по пройденной точке открывает крупную модалку поверх маршрута.

### Если у точки есть только запись

- открывается запись;
- табов нет.

### Если у точки есть только артефакт

- открывается карточка артефакта;
- табов нет.

### Если у точки есть и запись, и артефакт

- по умолчанию открывается `Запись`;
- внизу модалки есть переключатели:
  - `Запись`
  - `Артефакт`
- переключение происходит без перезагрузки модалки и без закрытия окна.

## Архив

### Переименование

`Дневник` переименовывается в `Архив`.

### Роль Архива

Архив больше не дублирует главный экран и не используется как первичное место reveal.

Его задача:

- перечитывать уже открытые записи;
- пересматривать найденные артефакты.

### Структура Архива

Архив делится на `2` вкладки:

- `Записи`
- `Артефакты`

### Вкладка `Записи`

- список открытых записей;
- записи показываются с портретом автора;
- запись открывается в крупной модалке полного текста.

### Вкладка `Артефакты`

- сетка найденных артефактов;
- по нажатию открывается большая карточка артефакта.

## Нижняя панель навигации

Панель всегда видна на главном экране маршрута.

Состав:

- `Архив`
- `Маршрут дня`
- `Настройки`

Формат:

- только иконка, без текста;
- все иконки рисуются в одном спокойном архивно-экспедиционном стиле;
- активный пункт выделяется мягкой тёплой подложкой / свечением;
- вся нижняя плашка читается как постоянный `tab bar`, а не как текстовый футер.
- нижняя плашка живёт как отдельная зона под игровым полем, а не внутри рамки листа.

Иконки:

- `Архив` — папка с вкладкой;
- `Маршрут дня` — простой компас;
- `Настройки` — простая шестерёнка.

## Правила

`Правила` убираются из постоянной нижней панели.

Новая логика:

- на экране пасьянса появляется иконка `?`;
- по нажатию открывается модальное окно с правилами пасьянса.

## Окно выигрыша

Окно выигрыша сохраняет текущий общий каркас.

Изменения:

- вместо слова `Запись` показывается портрет персонажа, от чьего лица идёт запись;
- вместо слова `Артефакт` показывается маленькое изображение артефакта;
- reveal-логика через `Вы нашли` сохраняется.

## Портреты персонажей

Портреты нужны для:

- модалки записи;
- `Архив > Записи`;
- окна выигрыша.

Список персонажей:

- Алексей Воронов
- Елена Мирская
- Георгий Левин
- Михаил Руденко
- Софья Климова

Для каждого персонажа нужен отдельный единый портретный мастер-ассет.

## Что меняется в продуктовой логике

### Было

- главный meta-экран = карта;
- записи и артефакты в основном читаются через Дневник;
- фрагменты карты — отдельный reveal-тип.

### Станет

- главный meta-экран = маршрутный лист;
- запись и артефакт читаются через точку маршрута;
- Архив хранит уже открытое;
- фрагменты карты как отдельный gameplay-тип уходят.

## Scope

Внутри этой задачи:

- новый главный экран `Маршрут`;
- новая модель листов маршрута;
- новый UX точки;
- переименование `Дневник` -> `Архив`;
- новая нижняя навигация;
- перенос `Правил` в экран партии;
- портреты персонажей как обязательный контентный слой;
- обновление окна выигрыша под портреты и мини-изображения артефактов.

Вне scope этого документа:

- финальные визуальные стили портретов;
- конкретная реализация свайпа и desktop-кнопок по коду;
- финальные тексты правил;
- полная переработка daily-mode beyond new entry point from bottom nav.

## Ожидаемый результат

- главный экран становится narrative-first и маршрутно-ориентированным;
- игрок воспринимает точки как реальные этапы экспедиции;
- Архив остаётся полезным, но не дублирует главный экран;
- UI становится чище и сильнее связан с историей пропавшей экспедиции.

## Detail-страница вместо модалок

Старые модалки точки убираются. Клик по пройденной точке теперь ведёт на отдельную detail-страницу, а не открывает overlay поверх маршрута.

Правила:

- если у точки только запись — открывается detail-страница типа `Запись`;
- если у точки только артефакт — открывается detail-страница типа `Артефакт`;
- если у точки есть и запись, и артефакт — по умолчанию открывается `Запись`, а внутри страницы есть переключение на `Артефакт`.

## Единый layout detail-страницы

Вместо набора modal-overlay вводится одна универсальная detail-страница, которая показывает либо запись, либо артефакт.

Принципы:

- тот же layout-каркас, что и у `Маршрута`, `Архива`, `Настроек`;
- та же нижняя плашка с тремя иконками;
- модального бордюра и отдельной рамки нет;
- расстояния между блоками сокращены;
- контент занимает максимум вертикали;
- на странице есть дополнительная кнопка `Домой`, ведущая на главный маршрут.
