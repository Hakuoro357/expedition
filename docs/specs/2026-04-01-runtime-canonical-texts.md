# Runtime Canonical Texts

## Статус

`Approved`

## Задача

Синхронизировать runtime-тексты с актуальным narrative-каноном.

Сейчас в игре:

- `entries.ru.ts` содержит пересказы;
- `Diary` и reward-модалка записи показывают summary вместо канонического текста;
- описания артефактов живут как отдельный runtime-набор и должны быть явно синхронизированы с финальным каноном.

## Решение

### 1. Записи

Для `ru` runtime-каноном становятся полные записи из:

- [STORY.md](C:\Users\RobotComp.ru\games\Yandex\01-solitaire-expedition\docs\specs\STORY.md)

Правило:

- `Diary` показывает полный текст записи;
- reward-модалка записи показывает полный текст записи;
- карточка записи в блоке `Вы нашли` показывает не полный текст, а короткий excerpt.

### 2. Excerpt для reward-карточки записи

Для записи на reward-экране нужен отдельный короткий текст.

Правило:

- excerpt делается из канонической записи;
- это 1 короткий фрагмент без потери смысла;
- excerpt не заменяет основной текст записи;
- excerpt используется только в компактной карточке `Вы нашли`.

### 3. Артефакты

Runtime-описания артефактов в:

- [artifacts.ts](C:\Users\RobotComp.ru\games\Yandex\01-solitaire-expedition\src\data\artifacts.ts)

должны быть синхронизированы с актуальным финальным набором описаний.

Правило:

- карточка артефакта на reward-экране и в `Diary` использует один и тот же канонический текст;
- в runtime не должно быть старых черновых описаний.

## Scope

В этой задаче обновляем:

- `entries.ru.ts`
- логику короткого excerpt для reward-карточки записи
- `artifacts.ts`

Вне scope:

- полная литературная переработка `entries.global.ts`
- новая схема локализации для non-`ru`
- изменение структуры narrative data beyond needed text fields

## Ожидаемый результат

- в `Diary` и reward-модалке игрок читает канонические записи, а не пересказ;
- на reward-карточке записи остаётся короткий и читаемый excerpt;
- описание артефакта везде совпадает с финальным каноном.
