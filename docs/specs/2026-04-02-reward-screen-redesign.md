# Reward Screen Redesign

## Статус

`Approved`

## Цель

Привести экран награды к новой продуктовой модели:

- без устаревшего reveal для фрагментов карты;
- без detail-модалок;
- с переходами на общую detail-страницу;
- с единым layout-принципом.

## Что меняется

Экран награды остаётся экраном результата точки, но обновляется под текущую структуру игры.

Показываются:

- монеты;
- прогресс;
- блок `Вы нашли`;
- кнопка продолжения;
- rewarded ad как вторичный сервисный блок.

## Новые типы reveal на reward-экране

Для текущей версии остаются только:

1. `Запись`
2. `Артефакт`

Reveal фрагментов карты как отдельного типа убирается.

## Карточки наград

Карточки в `Вы нашли` должны быть ближе к текущему narrative UX:

- для записи показывается портрет автора;
- для артефакта показывается маленькое изображение артефакта;
- карточки по клику не открывают модалку;
- карточки ведут на общую detail-страницу.

## Переходы

### Запись

Клик по карточке записи:

- открывает общую detail-страницу;
- по умолчанию в tab `Запись`.

### Артефакт

Клик по карточке артефакта:

- открывает общую detail-страницу;
- по умолчанию в tab `Артефакт`.

## Layout

Экран награды должен быть приведён к новой системе:

- без старой тяжёлой рамки;
- без старых modal detail overlay;
- с той же типографикой, что и в новом meta-UI;
- с более спокойной и цельной композицией.

## Что уходит

- reveal `map fragment`;
- отдельные detail-модалки наград;
- старый modal-style detail flow поверх reward-экрана.

## Вне scope

- изменение логики начисления монет;
- изменение rewarded ad механики;
- изменение порядка выдачи reward data в save/service-слое.
