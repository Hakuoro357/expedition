# Reward Reveal Flow

## Статус

`Approved`

## Задача

Исправить текущую проблему наград после прохождения точки:

- артефакты открываются незаметно и фактически "теряются" в дневнике;
- записи дневника открываются без момента первого чтения;
- фрагменты карты не имеют собственного понятного reveal-экрана;
- игрок может завершить точку и не понять, что именно получил.

## Продуктовый принцип

После прохождения точки игрок должен сразу видеть, что именно открылось.

`Дневник` и `Карта` после этого работают как архив уже открытого, а не как место первого reveal.

## Главный сценарий

После победы игрок попадает на `Экран результата точки`.

На экране есть:

- блок `Монеты`;
- блок `Прогресс главы`;
- блок `Вы нашли`.

В блоке `Вы нашли` показываются только новые награды текущей точки.

## Главные типы наград

Для `v1.0` reward UX строится вокруг трёх понятных для игрока типов:

1. `Артефакт`
2. `Запись дневника`
3. `Фрагмент карты`

Все прочие narrative-награды должны быть сведены к одному из этих трёх reveal-типов.

## Что считается открытием

### Артефакт

- открывается в момент завершения точки;
- сразу считается полученным;
- сразу становится доступен в дневнике;
- дневник не участвует в первом открытии.

### Запись дневника

- открывается в момент завершения точки;
- сразу считается открытой;
- сразу становится доступна в дневнике;
- дневник нужен для повторного чтения, а не для первого reveal.

### Фрагмент карты

- открывается в момент завершения точки;
- сразу считается полученным;
- основной экран карты после этого уже отражает новый прогресс;
- первое знакомство с фрагментом происходит не на карте, а через reveal на reward-экране.

## Структура экрана результата точки

### Верхняя часть

- короткий итог победы;
- монеты за точку;
- при необходимости бонус главы.

### Средняя часть

Блок `Вы нашли`:

- 1-3 новых награды текущей точки;
- каждая награда показана как отдельная карточка единой серии;
- тип награды отмечен маленьким бейджем/иконкой;
- у каждой карточки есть краткий title;
- игрок сразу видит полный состав награды этой точки.

### Нижняя часть

- прогресс главы;
- кнопка продолжения;
- дополнительные сервисные кнопки вроде rewarded ad, если они остаются в `v1.0`.

## Поведение блока `Вы нашли`

- блок показывает только новые награды этой точки;
- старые или уже ранее полученные награды туда не попадают;
- открывать награды подробно не обязательно;
- игрок может просто увидеть список наград и продолжить.

## Reveal-поведение по типам

### 1. Артефакт

По клику на иконку артефакта открывается модалка:

- название;
- большая картинка;
- короткое описание.

После закрытия игрок возвращается на экран результата точки.

### 2. Запись дневника

По клику на иконку записи открывается модалка записи:

- номер точки;
- автор;
- полный текст записи.

После закрытия игрок возвращается на экран результата точки.

### 3. Фрагмент карты

По клику на иконку карты открывается модалка `Карта обновлена`.

Она показывает не просто кусок бумаги, а изменение маршрута:

- мини-схему маршрута или текущего участка;
- новый фрагмент / новая метка выделены;
- короткая подпись объясняет, что именно добавилось.

Главная цель этого reveal:

- показать не предмет, а прирост понимания пути.

## Порядок наград внутри `Вы нашли`

Если на точке открывается несколько наград, порядок показа в блоке:

1. `Запись дневника`
2. `Артефакт`
3. `Фрагмент карты`

Причина:

- запись несёт основной narrative-контент;
- артефакт даёт предметный reveal;
- карта завершает пространственное понимание изменения маршрута.

## Роли экранов после изменения

### Экран результата точки

Первое и обязательное место reveal.

### Дневник

- повторное чтение записей;
- повторный просмотр артефактов;
- архив уже открытого.

### Карта

- накопленный прогресс маршрута;
- текущее состояние собранной структуры пути;
- не место первого reveal нового фрагмента.

## Визуальные решения

### Карточки в блоке `Вы нашли`

- все награды выглядят как единая серия карточек;
- тип различается маленьким бейджем/иконкой;
- отдельный визуальный язык для каждого типа награды не используется.

### Reveal карты

- карта открывается через модалку `Карта обновлена`;
- внутри используется мини-схема маршрута;
- новый участок или новая метка выделяются явно.

### Rewarded ad на экране результата

- rewarded ad остаётся на reward-экране;
- но живёт как вторичный сервисный элемент;
- блок `Вы нашли` и narrative reveal имеют приоритет;
- rewarded ad размещается ниже основного reveal-блока.

## Что нельзя допустить

- скрытого открытия артефактов только через дневник;
- скрытого открытия записей без первого момента чтения;
- ситуации, где игрок получил награду, но не понял, какую;
- reward-экрана, который показывает только монеты и не показывает narrative-результат точки;
- отдельного UX для десятков технических reward-типов вместо трёх понятных категорий.

## Решения для реализации

### Обязательно

- reward-экран должен уметь показывать несколько новых наград одной точки;
- каждая награда должна иметь иконку в `Вы нашли`;
- каждая иконка должна открывать подробную модалку;
- продолжение без открытия модалок должно быть допустимо.

### Не требуется для `v1.0`

- принудительно открывать все награды перед продолжением;
- делать отдельный reveal-экран на каждый reward subtype;
- переносить момент первого reveal в дневник или на основной экран карты.

## Влияние на текущую модель данных

Текущая модель данных в целом подходит.

Нужно не изобретать новые классы наград, а правильно отобразить уже существующие narrative rewards через три игроковых reveal-типа:

- `artifact`
- `entry`
- `map`

## Решённые продуктовые решения

- блок `Вы нашли` использует единый стиль карточек;
- тип награды маркируется маленьким бейджем/иконкой;
- карта открывается через модалку `Карта обновлена` с мини-схемой;
- rewarded ad остаётся на reward-экране, но как вторичный сервисный элемент ниже основного reveal-блока.
