# Solitaire: Expedition — Localization Plan

## Статус

`Зафиксировано как языковой план и правила локализации для контента и ассетов`

## Базовый принцип

- У игры должна быть `одна основная история` для всех языков.
- Для других языков не нужно придумывать новый сюжет, новых персонажей или новую мотивацию.
- Локализуется подача: тексты, описания, UI, store-материалы и narrative-фрагменты.
- Лор, структура глав, экспедиция и финальная тайна остаются едиными.

## Что переводится

- UI-тексты;
- описания игры в каталоге;
- how to play;
- названия глав;
- названия находок и артефактов;
- страницы дневника;
- подписи карты;
- тексты наград;
- промо-тексты для публикации.

## Что не переписывается заново для каждого языка

- базовая история;
- состав экспедиции;
- главный артефакт;
- структура глав;
- мотивация игрока;
- основная интрига и финальная разгадка.

## Правило имён персонажей

- Принят `вариант B`.
- В `ru` локали используется русский набор имён.
- Во всех `non-ru` локалях используется единый international-набор имён.
- Не делать отдельный набор имён под `en`, `de`, `fr`, `tr` и другие языки.
- Во всех текстах использовать стабильные `character_id`, а отображаемое имя брать из локали.

## Приоритет языков для Яндекс Игр

### Wave 1

- `ru` — обязательный базовый язык.
- `en` — обязательный базовый язык для внешнего трафика.
- `tr` — высокий приоритет.
- `zh` — высокий приоритет.
- `hi` — высокий приоритет.
- `vi` — высокий приоритет.

### Wave 2

- `es`
- `pt`
- `ar`
- `id`

### Wave 3

- `fr`
- `de`
- `ja`
- `it`
- `ko` — опционально, если платформа и производство подтверждают приоритет.

## Почему такой порядок

- По документации Яндекс Игр `ru` и `en` рекомендуются как базовый минимум.
- В официальных материалах платформы отдельно выделяются языки, которые могут дать дополнительный трафик.
- Для `v1.0` разумно брать языки волнами, а не пытаться сразу локализовать всё.

## Правила для ассетов

- Ключевые графические ассеты должны быть по возможности `без текста`.
- Если текст вшит в картинку, его придётся локализовать для каждого заявленного языка.
- Иконка игры должна быть без текста.
- Основная обложка должна быть либо без текста, либо с текстом, который легко заменить по языкам.
- Внутриигровые сюжетные экраны лучше собирать из фонового изображения и отдельного текстового слоя.
- Для дневника, карт и наград не вшивать длинные абзацы в растровые изображения.

## Что важно для production

- Делать тексто-независимые фоны.
- Отдельно хранить UI-плашки и текст.
- Для дневника и карты использовать универсальные декоративные шаблоны.
- Все имена файлов держать на латинице без кириллицы.
- Закладывать запас по длине строк для `de`, `fr`, `pt` и `es`.

## Ограничения по narrative-локализации

- Сохранять спокойный, архивный, взрослый тон.
- Не переводить историю в пафосное приключение.
- Не усиливать мистику там, где её нет в оригинале.
- Избегать слишком разговорной или комедийной интонации.
- Сохранять интригу и недосказанность до финала.

## Практический вывод для ассетов v1.0

- Сначала готовить ассеты так, чтобы они одинаково работали для всех языков.
- Минимизировать текст внутри графики.
- Сюжетные тексты хранить отдельно от изображений.
- Имена глав, находок и дневниковых записей локализовать как текстовый контент, а не как часть арта.
